# Configuration

Codernic is designed to be largely zero-configuration, but it allows powerful customizations via its configuration files and environment variables.

## Global Configuration

The global configuration is typically stored in `~/.config/codernic/config.toml` on Linux/macOS, or the equivalent AppData folder on Windows.

### Bring Your Own Key (BYOK) Management

Codernic supports "Bring Your Own Key" (BYOK) for both cloud-only and hybrid deployments. You can securely manage your keys for major providers (OpenAI, Anthropic, OpenRouter) to power reasoning engines while keeping your codebase strictly local.

Keys can be configured globally in your environment, or more securely within a provider configuration file (e.g., `openai.provider.json`) generated by the CLI.

```bash
export OPENAI_API_KEY="sk-..."
export ANTHROPIC_API_KEY="sk-ant-..."
```

### Deployment Strategies

Codernic supports multiple operational modes configurable in your `.codernic/config.toml` or workspace setup:
- **Local Only**: 100% local processing (e.g. using GGUF models downloaded via the CLI).
- **Hybrid (Local + Cloud)**: Local embeddings and indexing combined with Cloud API reasoning via BYOK. This maximizes privacy for code structure while leveraging powerful cloud LLMs for complex logic.
- **Cloud Only**: Fully managed cloud APIs for both reasoning and embeddings (BYOK).

## Workspace Configuration

Codernic inspects the local directory for a `.agencee` or `.codernic` configuration folder to override global settings.

Create a `.codernic/config.toml` in your project root to define workspace-specific settings:

```toml
[project]
name = "my-awesome-app"
strict_mode = true

[engines]
default_model = "claude-3-5-sonnet"
max_context_tokens = 128000
```

## MCP (Model Context Protocol) Bridge

Codernic uses the MCP Bridge to seamlessly communicate between local IDEs and the AI engines. You can configure custom tools and server integrations in the `mcp.json` file inside your workspace configuration.
